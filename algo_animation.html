02 — Reference

Graph Traversal SEARCH

  • BFS — Explores layer-by-layer using a queue. Finds shortest path in unweighted graphs. Visits all nodes at distance d before distance d+1.
  • DFS — Explores as deep as possible before backtracking. Uses a stack (or recursion). Basis for cycle detection, SCC, topological sort.
  • Topological Sort — Linear ordering of vertices in a DAG such that for every edge u→v, u comes before v. Kahn's algorithm uses in-degree counting.

Single-Source SP SSSP

  • Dijkstra — Greedily picks the unvisited node with minimum distance. Requires non-negative weights. O((V+E) log V) with a min-heap.
  • Bellman-Ford — Relaxes all edges V−1 times. Handles negative weights and detects negative cycles. Slower but more general than Dijkstra.

All-Pairs SP APSP

  • Floyd-Warshall — Dynamic programming approach. For each intermediate vertex k, tries to improve dist[i][j] via i→k→j. Works with negative weights but not negative cycles.
  • The distance matrix converges to the true all-pairs shortest path after N iterations over intermediate vertices.

Minimum Spanning Tree MST

  • Prim's — Grows the MST from a seed node. Always picks the cheapest edge crossing the cut. Efficient on dense graphs with an adjacency matrix.
  • Kruskal's — Sorts all edges, then greedily adds the cheapest edge that doesn't form a cycle. Uses Union-Find. Efficient on sparse graphs.
  • Both produce the same MST weight, but possibly different trees for tied edges.

Network Flow MAX-FLOW

  • Ford-Fulkerson — Iteratively finds augmenting paths from source S to sink T in the residual graph and pushes flow along them. Terminates when no augmenting path exists.
  • Edmonds-Karp (this demo) — Ford-Fulkerson using BFS to find the shortest augmenting path. Guarantees O(VE²) time regardless of edge capacities.
  • Max-Flow Min-Cut Theorem — The maximum flow equals the minimum cut capacity. When the algorithm terminates, the reachable set from S defines the minimum cut.
  • Residual edges enable flow cancellation: sending flow on u→v creates capacity on v→u in the residual graph, allowing the algorithm to "undo" suboptimal decisions.
